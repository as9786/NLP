# Training Compute-Optimal Large Language Model

## 초록
- The number of training tokens and the model parameter size should be scaled up at the same rate
- $N_{opt}(C), D_{opt} (C) = argmin L(N,D)$
- Their goal is to find the optima number parameters and training tokens that can be allocated under a given compute budget

## 관련 연구

### 멱 법칙(Power-Law relationship)
- 가중치 수와 성능 사잉의 연관관계
- $Loss \propto N^{-\alpha}$
- 본 논문은 해당 관계에서 주어진 예산을 추가하는 것이 목표

## 방법
- Estimating the optimal parameter/training tokens allocation
- 가중치 수 조정
- Adjustment of the number of training tokens
- 그에 따른 성능 곡선 확인. 분석하여 경험적으로 최적화

### 접근 1
- FIx model sizes and vary number of training tokens
- 모형 : 70M ~ 10B
- For each model, four different training sequences are prepared
- Training sequence
    - 학습율 실험 (10배씩 감소)
    - Training token experiments (Across up to a 16x range)

- 정의역이 비는 부분들은 선형 보간법을 통해 치역값을 채움
- $N \propto C^a \ and \ D \propto C^b$
- $a \approx 0.5 \, \ b \approx 0.5$
- In other words, the model parameter size and the number of training tokens should be scaled up at the same rate

### Approach 2 : IsoFLIP profiles
- Experiments were conducted by varying the model size across nine different training otken budgets
- 주어진 훈련 예산에서 최적의 가중치 수를 직접적으로 찾을 수 있도록 진행하는 실험
- a=0.49, b=0.51
- 첫번째 접근과 동일한 결론

### 접근 3
- Fitting a parametric loss function
- $ \hat L (N,D) = E + \frac{A}{N^{\alpha}} + \frac{B}{D^{\beta}}$
- E : 기본적인 손실
- $A/N^{\alpha}$ : 모형 가중치가 부족해서 생기는 오차항
- $B/D^{\beta}$ : The error term arising from limited training tokens$
- a=0.46, b=0.54
